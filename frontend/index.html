💬 Frequently Asked Questions
BlockVerify is a custom blockchain system that lets you register AI models and verify their integrity over time. Every registration creates an immutable record on our blockchain — if someone tampers with the model file, the hash won't match and you'll know immediately.
No account needed. Just type any username in the top-right field and start using the app. Your username is used as the owner tag for your models — pick something memorable so you can find your models later in the My Models tab.
Any file — .pt, .h5, .pkl, .onnx, .bin, PDFs, ZIPs, etc. The app only reads the file to compute its SHA-256 hash. The file itself is never uploaded to any server — all hashing happens in your browser.
When you register a model, a unique 16-character Model ID is generated (e.g. a3f9b2c1d4e8f7a0). It is automatically copied to your clipboard. You can also find all your Model IDs in the My Models tab. Keep this ID — you need it for verification and version history.
Go to the Verify tab → upload the model file → paste the Model ID → click Verify. If the SHA-256 hash of your file matches what's stored on the blockchain, it's verified as intact. If it doesn't match, the model has been modified since registration.
Each write operation (register, verify, add version) creates a new blockchain block. Mining means finding a nonce value that makes the block's SHA-256 hash start with 0000. This takes thousands of attempts and makes the chain computationally expensive to tamper with. You can see mining time and attempts in the ⛏ badge after each operation.
It's a safe demo in the Explorer tab. The backend temporarily modifies a block's data, runs the chain validator (which detects the change), then immediately restores the original data. Nothing is permanently changed — it just proves your blockchain can detect tampering.
Each username can perform at most 10 write operations (register, verify, add version) per 60 seconds. This prevents the blockchain from being flooded. If you hit the limit, wait a moment and try again — the Register tab shows your remaining quota after each operation.
The backend server may not be running. Make sure you started it with python backend/app.py from the project root. If using the hosted version, the Render free tier may be sleeping — wait 30 seconds and try again after the first request wakes it up.
All data is persisted to JSON files in backend/data/ on every write. If the server restarts, data is reloaded automatically. On Render's free tier, the disk is ephemeral — data resets on each deploy. For production use, attach a persistent disk.
